# ФАЗА 1.3: Создание дизайн-агента

## Цель
Создать специализированного агента для помощи в разработке UI компонентов.

## Задачи

### 1. Создание агента `ui-designer`
- [ ] Использовать скилл `agent-development`
- [ ] Определить роль агента: эксперт по React и Tailwind CSS
- [ ] Настроить system prompt с лучшими практиками frontend

### 2. Конфигурация агента
**Frontmatter:**
```yaml
name: ui-designer
description: Use this agent when creating UI components, pages, or styling frontend elements
model: inherit
color: cyan
tools: ["Read", "Write", "Grep", "Glob"]
```

### 3. System prompt агента
**Основные обязанности:**
- Генерация чистого, переиспользуемого кода компонентов
- Следование принципам доступности (a11y)
- Использование Tailwind CSS для стилизации
- Избегание generic "AI aesthetics"

### 4. Интеграция скиллов
- [ ] Добавить скилл `frontend-design` для креативного подхода
- [ ] Добавить ссылки на `design-lab` для итеративного дизайна

## Ожидаемый результат
- Рабочий агент `ui-designer.md` в папке `agents/`
- Агент способен генерировать качественные UI компоненты

## Примеры использования
```
Context: Нужно создать компонент таблицы заказов
user: "Создай компонент таблицы заказов с сортировкой и пагинацией"
assistant: [Загружает агента ui-designer для генерации кода]
```
