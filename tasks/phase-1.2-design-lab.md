# ФАЗА 1.2: Дизайн-сессия с Design Lab

## Цель
Создать привлекательный и функциональный дизайн для всех страниц приложения через итеративный процесс с Design Lab.

## Задачи

### 1. Подготовка к дизайн-сессии
- [ ] Запустить dev-сервер для frontend
- [ ] Подготовить описание для каждой страницы

### 2. Дизайн главной страницы (Dashboard)
- [ ] Запустить `/design:start Dashboard`
- [ ] Пройти интервью с агентом Design Lab
- [ ] Просмотреть 5 вариантов в `http://localhost:3000/__design_lab`
- [ ] Выбрать лучший вариант или синтезировать из нескольких
- [ ] Повторить итерации при необходимости

### 3. Дизайн страницы заказов
- [ ] Запустить `/design:start OrdersPage`
- [ ] Описать требования: таблица, фильтры, формы CRUD
- [ ] Выбрать финальный дизайн

### 4. Дизайн страницы пользователей
- [ ] Запустить `/design:start UsersPage`
- [ ] Описать требования: таблица, уровни доступа, поиск
- [ ] Выбрать финальный дизайн

### 5. Дизайн страницы авторизации
- [ ] Запустить `/design:start AuthPage`
- [ ] Описать требования: формы входа и регистрации
- [ ] Выбрать финальный дизайн

### 6. Финализация
- [ ] Выполнить `/design:cleanup` при необходимости
- [ ] Проверить сгенерированный `DESIGN_PLAN.md`
- [ ] Проверить сгенерированный `DESIGN_MEMORY.md`

## Ожидаемый результат
- 4 финальных дизайна для основных страниц
- Файл `DESIGN_PLAN.md` с планом реализации
- Файл `DESIGN_MEMORY.md` с запомненными стилевыми решениями

## Команды Design Lab
```bash
/design:start [ComponentName]  # Начать сессию
/design:cleanup                # Очистить временные файлы
```

## Примечания
- Держать dev-сервер запущенным во время сессии
- Быть конкретным в ответах на вопросы интервью
- Ссылаться на известные продукты для вдохновения (Linear, Stripe, etc.)
