# Общая задача на проект.

1. Разработать полнофункциональное веб-приложение на Fast-Api для управления заказами в кафе. Приложение должно
   позволять добавлять, удалять, искать, изменять и отображать заказы, через Веб-интерфейс.
2. Также должно быть реализована аутентификация пользователей и управление пользователями (добавлять, удалять, искать,
   изменять, отображать пользователей в соответствии с правами текущего пользователя).
3. Дополнительно, предоставить API для работы с заказами и пользователями (добавление, удаление, поиск и т. д.).

## Каждый заказ должен содержать следующие поля:

- id (уникальный идентификатор, генерируется автоматически)
- table_number (номер стола)
- items (список заказанных блюд с ценами)
- total_price (общая стоимость заказа, вычисляется автоматически)
- status (статус заказа: “в ожидании”, “готово”, “оплачено”)

## Каждый пользователь должен содержать следующие поля:

- id (уникальный идентификатор, генерируется автоматически)
- nikname (никнейм аккаунта)
- name (Имя)
- surname (Фамилия)
- email (адрес электронной почты)
- level (уровень прав (доступа), например: "гость", "клиент", "сотрудник", "менеджер", "администратор", "
  руководитель", "суперпользователь")
- status (статус аккаунта: активный, пассивный)

## Функциональные требования к заказам:

1. Добавление заказа:

- Через веб-интерфейс пользователь вводит номер стола и список блюд с ценами. Система автоматически добавляет заказ с
  уникальным ID, рассчитанной стоимостью и статусом “в ожидании”.

2. Удаление заказа:

- Пользователь через веб-интерфейс выбирает заказ по ID и удаляет его из системы.

3. Поиск заказа:

- Возможность поиска заказов по номеру стола или статусу через поисковую строку.

4. Отображение всех заказов:

- Веб-страница с таблицей всех заказов, отображающая их ID, номер стола, список блюд, общую стоимость и статус.

5. Изменение статуса заказа:

- Пользователь через интерфейс выбирает заказ по ID и изменяет его статус (“в ожидании”, “готово”, “оплачено”).

6. Расчет выручки за смену:

- Отдельная страница или модуль для расчета общего объема выручки за заказы со статусом “оплачено”.

## Функциональные требования к пользователям:

1. Добавление пользователя:

- Через веб-интерфейс пользователь регистрируется и добавляется в БД с предоставлением уровня "клиент". Все
  незарегистрированные пользователи, автоматически приравниваются к уровню "гость".
- Через веб-интерфейс пользователь с уровнем прав администратор и выше создает/добавляет нового пользователя.

2. Удаление пользователя:

- Пользователь через веб-интерфейс из своего личного кабинета может удалить свой аккаунт.
- Пользователь уровнем прав администратор и выше через веб-интерфейс может удалить свой аккаунт и любой другой ниже
  уровнем (уровнями).

3. Поиск пользователя:

- Возможность поиска пользователем уровнем прав менеджер и выше другого пользователя ниже уровнем по ID и/или (Имени и
  Фамилии) через поисковую строку.

4. Отображение всех пользователей:

- Веб-страница с таблицей всех пользователей, отображающая их ID, Имя, Фамилию, Почту, Уровень. Поиск возможен только
  пользователей уровнем (уровнями) ниже.

5. Изменение пользователя:

- Пользователь с уровнем прав администратор и выше через интерфейс выбирает пользователя уровнями ниже по ID и изменяет
  его уровень (но всегда только ниже своего), Имя, Фамилию, Почту и статус. Понизить любого пользователя до уровня "
  гость" нельзя.
